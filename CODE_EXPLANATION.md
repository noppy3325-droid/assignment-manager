# アプリケーション コード解説書

本ドキュメントは、このタスク・スケジュール管理アプリケーションの設計思想、アーキテクチャ、ファイル構造、および実装の詳細について解説するものです。

## 1. アプリケーションの概要

本アプリケーションは、学生や社会人が日々の課題（タスク）や予定を美しく・効率的に管理するための「タスク＆スケジュール管理アプリ」です。
Firebaseを用いたリアルタイムなデータ保存、Pomodoro（ポモドーロ）タイマー機能、自由な科目（タグ）管理、そして直感的なドラッグ＆ドロップ機能を備えています。

## 2. 主要な技術スタック

- **フロントエンド・フレームワーク**: React 19, TypeScript, Vite
- **スタイリング**: Tailwind CSS v4, カスタムテーマ（Material Design 3ベースのCSS変数を使用）
- **アニメーション**: `motion` (旧Framer Motion) による流体的なアニメーション
- **データベース & 認証**: Firebase (Authentication: Google ログイン, Cloud Firestore: データ保存)
- **アイコン**: `lucide-react`
- **ドラッグ＆ドロップ**: `@dnd-kit/core` ほか関連パッケージ
- **日付処理・カレンダー**: `date-fns`, `react-day-picker`
- **エフェクト**: `canvas-confetti`（タスク完了時の紙吹雪演出）

## 4. コア機能とコードの詳細

### 4.1. データモデリング (`types.ts`)
アプリケーションの中心的なデータ構造は `Submission`（タスク・課題）です。
- タスクはパーセンテージ（`percentage`）またはページ単位（`pages`）で進捗を管理できます。
- 優先度 (`priority`)、削除フラグ（ごみ箱機能 `isDeleted`）、完了フラグなどを持っています。
- サブコレクション的に、作業ログを記録するための `activityLogs` を持っています。

### 4.2. UIと状態管理 (`App.tsx`)

`App.tsx` には多くのUIとロジックが集約されています。主に使用されているHooksについて：

- **`useState`**: カレンダーの表示/非表示 (`isCalendarOpen`)、モーダル状態 (`isAdding`, `selectedId`)、テーマ設定等の状態。
- **`useEffect`**: Firestoreを使ったデータの購読（`onSnapshot`で `submissions` をリアルタイム監視）。ブラウザを閉じた時や非表示になった時に設定を保存するバックグラウンド同期 (`visibilitychange`, `beforeunload`イベント)。

### 4.3. Firebaseとの連携
- **認証 (`login`, `logout`)**: GoogleAuthProviderを用いたポップアップログインを利用しています。
- **データ監視 (`onSnapshot`)**: ログイン中のユーザーのUIDをキーに、`submissions` コレクションをフェッチし、タスクの変更があった瞬間にフロントエンドも再描画されます。

### 4.4. ドラッグ＆ドロップ (`@dnd-kit/core`)
- カレンダーへのタスク配置や、UI上でのタスクの並び替え用に `DndContext` が用いられています。
- 予定日の変更などを直感的に操作できる設計になっています。

### 4.5. タイマーと進捗管理 (ポモドーロ機能)
- `useInterval`ライクな実装を用いて、アクティビティごとに作業時間を測ります。（変数: `activeTimerId`, `timerSeconds` 等）
- 作業時間が終わると進捗入力画面（`showProgressInput`）が表示され、`finalizeActivity` メソッドでFirestoreに進捗とログ（`activityLogs`）を記録します。

### 4.6. カレンダー機能
- カスタムで組み上げられたカレンダーUI（月間・週間・年間切り替え）があり、タスクの「締切日（Deadline）」だけでなく「カレンダー展開用の予定日（Scheduled Date）」を視覚的にマッピングしています。

### 4.7. アニメーションと完了エフェクト
ユーザー体験を向上させるために以下のリッチな演出が施されています：
- **`motion` によるトランジション**: 画面の遷移、モーダルの展開 (`LayoutGroup`, `AnimatePresence`)。
- **完了時の演出**: タスクが100%（完了）に達した際に `canvas-confetti` による紙吹雪のエフェクトが実行されます。
- **「済」スタンプ**: `SubmissionCard` コンポーネント内に仕込まれており、`status === 'completed'` の際に `motion.div` を使って跳ね回って配置される赤いハンコの演出（DONE / 済）を表現しています。背景色も自動でグレースケールになるように `cn()` ヘルパーでTailwindクラスが制御されています。

### 4.8. カスタマイズと多言語対応
- 設定モーダルからテーマ（デフォルト, ダーク, フラワー, 犬, 猫...など）を変更可能で、アイコン等も条件分岐で変わる遊び心が実装されています。
- ユーザーに合わせた言語設定 (`translations.ts`) がFirebaseの `users` ドキュメントに保存され、別の端末でも同じ設定・言語状況でアプリを開けるように設計されています。

## 5. アプリケーションの特長と設計思想

- **M3 (Material Design 3) ライクなデザイン**: CSSのカスタムプロパティ（`--m3-surface`, `--m3-primary` 等）を豊富に定義し、`index.css` で全体の統一感（配色、丸み）を作っています。
- **オプティミスティックUI風のレスポンス向上**: データを Firestore へ送る前にも、フロントエンド側のアニメーションが滑らかに動くよう考慮されています。
- **アクセシビリティ (`Keyboard Shortcuts`)**: `/` キーでの検索、`c` でのカレンダーオープン、`j/k` でタスク選択など、キーボードだけでもある程度の作業がこなせるショートカットが実装されています。
- **セキュアな設計**: Firebase セキュリティルールによって、自分のUIDを持つドキュメントしか作成・読み書きできない安全な運用になっています。
